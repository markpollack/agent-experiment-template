## Process

1. **Read first** — Read the project structure, build config, and all source code before making changes.
2. **Plan** — Identify what needs to change and in what order.
3. **Implement** — Make changes incrementally, verifying each step compiles.
4. **Verify** — Run `./mvnw clean test` to confirm everything passes.
5. **Review** — Check your changes for correctness, edge cases, and style consistency.

## Constraints

- Do NOT introduce breaking changes to existing functionality
- Follow the project's existing code style and conventions
- Commit only working code — verify before finishing
